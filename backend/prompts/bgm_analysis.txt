## 重要：你无法访问音频文件
你只能基于下方提供的**客观音频特征**（由 librosa 提取）进行分析。
**不要假装自己听了这首歌。**
**所有判断必须严格基于给出的客观特征，不得自己臆测。**

## 客观音频特征（不可推翻，必须遵守）
- BPM：**{bpm}**（这是 librosa.beat_track 的准确结果，不可质疑）
- 能量均值：{energy_mean}（RMS 均值，0-1）
- 能量变异系数：{energy_cv}（能量波动程度，>0.3 表示波动大）
- 前后能量比：{energy_ratio}（后半段能量 / 前半段能量）
- 频谱质心：{spectral_centroid} Hz（音色亮度）
- 零交叉率：{zcr}（高频成分多少）
- Onset 密度：{onset_density}（节奏感强弱）
- 高潮段位置：{climax_start}s ~ {climax_end}s
- 结构边界（秒）：{structure_bounds}
- 配器推断：{instruments}
- 时长：{duration}s

## 分析规则（必须严格遵守）

### 1. 情绪打分规则（0-10 整数）
- **必须和客观特征一致**：
  - 如果 energy_cv > 0.4 且 energy_ratio > 1.5 → tension 和 epic 至少 6 分
  - 如果 energy_mean < 0.15 且 energy_cv < 0.2 → calm 至少 7 分，epic 最多 3 分
  - 如果 BPM > 130 且 energy_mean > 0.2 → calm 最多 3 分
  - 如果 spectral_centroid > 3000 → brightness 高，适合电子/流行，joy 或 excitement 至少 5 分
  - 如果 spectral_centroid < 1500 → 低频为主，适合 hip-hop/电子，mystery 或 calm 可能较高
- **允许矛盾情绪同时存在**（现实中的音乐是复杂的）：
  - joy + sadness 可以同时 >= 5（苦乐参半）
  - calm + tension 可以同时 >= 5（平静但紧张，如悬疑片）
  - 但 **epic + calm 不能同时 >= 7**（壮丽和平静是对立的）
- **打分分布要求**：
  - 至少 2 个维度 >= 6
  - 至少 2 个维度 <= 3
  - 所有维度之和 **必须在 20-35 之间**（不再是死板的 25）

### 2. 情绪弧线规则（必须基于 energy_cv 和 energy_ratio）
- 如果 energy_cv < 0.25 → **必须选"平稳型"**
- 如果 energy_cv >= 0.25 且 energy_ratio > 1.5 → **必须选"山峰型"或"上升型"**
- 如果 energy_cv >= 0.25 且 energy_ratio < 0.7 → **必须选"下降型"**
- 如果 energy_cv >= 0.4 → **必须选"波浪型"或"山峰型"**
- **不允许违反上述规则**

### 3. 节奏标签规则
- 如果 BPM < 90 → 可以包含"缓慢铺底""无鼓点"
- 如果 BPM >= 120 → **必须包含**"固定节拍"或"四四拍强驱"或"循环律动"
- 如果 onset_density > 0.5 → **必须包含**"军鼓驱动"或"808贝斯驱动"（看配器）
- **不允许**同时选"无鼓点"和"BPM >= 120"

### 4. 适合/不适合场景规则
- **必须和情绪弧线、BPM、能量一致**：
  - 如果 BPM > 120 且 energy_mean > 0.2 → **不能**把"冥想瑜伽"放进 fit_scenes
  - 如果 energy_cv < 0.2 → **可以**把"日常Vlog""冥想瑜伽"放进 fit_scenes
  - 如果 climax_start > 0 → **可以**把"运动健身""舞蹈"放进 fit_scenes
- fit_scenes 和 unfit_scenes **不能有交集**
- **至少 3 个 fit_scenes，至少 2 个 unfit_scenes**

### 5. 起始位置规则
- 如果 intro 很长（structure_bounds[1] > 15 秒）→ start_position_sec = structure_bounds[1]（跳过 intro）
- 如果 高潮段很明显 → start_position_sec = max(0, climax_start - 5)（高潮前 5 秒进入）
- 否则 → start_position_sec = 0
